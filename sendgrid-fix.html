🔧 SendGrid Sender Identity Fix

❌ Error Detected:

"The from address does not match a verified Sender Identity"

Your FROM_EMAIL environment variable doesn't match your verified SendGrid sender email.

📋 Step-by-Step Fix

1. Check Your SendGrid Verified Sender

Go to your SendGrid Dashboard:

SendGrid Dashboard → Settings → Sender Authentication → Single Sender Verification

Copy the exact email address that shows as "Verified" ✅

2. Update Supabase Environment Variable

Go to your Supabase Dashboard and update the FROM_EMAIL variable:

Supabase Dashboard → Settings → Edge Functions → Environment Variables Variable: FROM_EMAIL Value: your-verified-sendgrid-email@domain.com

3. Redeploy Functions

Environment variables only take effect after redeployment:

supabase functions deploy send-supplier-docs supabase functions deploy send-freight-query-email supabase functions deploy send-sowing-alerts

4. Test the Fix

Try sending an email again. It should work now!

⚠️ Common Issues:

  • Make sure the email is exactly the same (case-sensitive)
  • Don't forget to redeploy after changing environment variables
  • Verify the sender in SendGrid first before using it
  • Check for typos in the email address

✅ Alternative Solutions:

If you don't have a verified sender yet:

  1. Go to SendGrid → Settings → Sender Authentication
  2. Click "Create New Sender" in Single Sender Verification
  3. Add your desired email address
  4. Check your email and click the verification link
  5. Update the FROM_EMAIL variable with the verified email
  6. Redeploy the functions
